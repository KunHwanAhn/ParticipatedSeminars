# Clean code 101
- 강경완

## What is clean code for us?
- 일단은 지금 짜고 `나중에` 고치자 -> `나중`은 결코 오지 않는다.
- 미리 좋은 습관을 들여서 작성해야 한다.

## How to do clean code?
- 성능부터 올려보자. (알고리즘, 대용량 데이터처리를 제외하고...)
  - Java 기준, String VS StringBuilder
     - String은 할당될 때마다, 계속 메모리를 증가시키기에 GC가 자주 일어나서 성능을 저하시킬 수 있음
     - StringBuilder를 사용해라.
- 내 코드는 나만 읽지 않는다.
  - 의미있는 이름을 지어라.
  - 이름을 짧게 짓고 주석을 하기보다는, 길게 짓더라도 한 눈에 읽고 이해할 수 있는 이름으로
  - 클래스 이름은 명사 / 메서드 이름은 동사 => 주석 대신 코드로 Communication을
- 주석은 What을 표현하는 것이 아니라 How를 표현해야 한다.
  - 주석은 코드로 표현하기 어려운 `의도`를 표현하기 위해서 사용해라.
- 1개의 함수에는 1개의 기능
- 최대한 작게
- 하나만 추상화, 그리고 의도를 서술하라
- 확실한 try-catch()
